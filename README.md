# Azure IAM Automation Portfolio
This repository represents my comprehensive, hands-on work in Azure Identity & Access Management (IAM), built to demonstrate real-world engineering capability across authentication, authorization, lifecycle automation, privileged access, threat detection, and Zero Trust enforcement. The goal of this portfolio is to show not only that I can write scripts, but that I understand the architecture, governance, and security principles behind modern identity systems. Every folder, script, and diagram in this repository reflects the responsibilities of an IAM engineer working in a cloud-first, security-driven environment.

Identity is the foundation of Zero Trust, and organizations rely on IAM engineers to ensure that the right people have the right access at the right time — and that this access is continuously monitored, governed, and secured. This repository demonstrates my ability to automate identity operations using Microsoft Graph, PowerShell, and Entra ID, while also documenting the workflows, decisions, and governance considerations that accompany each identity function. My work here is structured to mirror real enterprise IAM practices, including feature isolation, branching, pull requests, documentation, and architectural diagrams.

The repository is divided into six major IAM domains, each representing a core function within an enterprise identity program. The MFA Audit section focuses on strengthening authentication security by identifying users without MFA, detecting weak MFA methods, and exporting MFA registration data for audits. The Conditional Access Governance section provides automation for backing up, exporting, and comparing Conditional Access policies — a critical part of Zero Trust enforcement and change management. The Identity Lifecycle Automation section covers the Joiner–Mover–Leaver process, including provisioning, deprovisioning, license assignment, and guest account cleanup, all driven by HR data and Graph API automation.

Privileged access is one of the highest-risk areas in IAM, and the PIM Privileged Access section demonstrates my ability to audit privileged roles, detect standing admin assignments, and support Just-In-Time (JIT) access workflows. The Identity Threat Detection section focuses on analyzing sign-in logs to detect MFA fatigue attacks, impossible travel, brute-force attempts, and other identity-based threats. Finally, the Zero Trust Device Compliance section integrates identity with device posture by auditing device compliance, detecting missing BitLocker encryption, and supporting Conditional Access enforcement based on device trust.
